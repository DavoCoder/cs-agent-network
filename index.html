3. Architecture Summary

üìÑ View full documentation ‚Üí

Design Pattern: Dispatcher ‚Üí Specialized Agents

Agent Purpose Integration
Supervisor Classifies and routes tickets Structured classification (Pydantic schemas)
Technical Support Handles developer questions MCP (Model Context Protocol) for documentation
Billing Answers billing questions Pinecone vector store knowledge base
Administration Manages admin requests A2A protocol with mandatory human review

Key Architectural Choices

  • Explicit Graph Construction: Fine-grained control over execution flow, custom patterns (ReAct loops, interrupts)
  • Structured Output: Pydantic schemas ensure type-safe, consistent LLM outputs
  • Runtime Configuration: RunnableConfig allows model/prompt overrides without code changes
  • Shared State: ConversationState preserves context across agent transitions
  • Human-in-the-Loop: Mandatory interrupt pattern for regulatory compliance
  • Tool Isolation: Separate tool execution nodes per domain for clear boundaries

Quick Start

A well-structured multi-agent system built with LangGraph that intelligently routes and handles customer support tickets across specialized domains with human-in-the-loop capabilities.

Technology Stack

Technology Purpose
LangGraph Agent orchestration and graph execution
Python 3.11+ Core language (tested with Python 3.13)
uv Package manager and virtual environment
Pinecone Vector database for knowledge base
A2A Protocol Agent-to-Agent communication for admin operations
MCP Model Context Protocol for documentation access

Getting Started

Prerequisites

  • Python 3.11+
  • Install uv package manager
  • API keys: OpenAI, LangSmith, Pinecone
  • Optional: A2A server for admin agent

Quick Commands

  • uv sync - Install dependencies
  • uv run langgraph dev - Start server
  • uv run pytest tests/unit/ - Run tests
  • uv run python vector_db/seed_vector_kb.py - Seed KB

Architecture Pattern

The system uses a dispatcher ‚Üí specialized agents architecture:

  • Supervisor: Routes tickets to appropriate agents
  • Technical Support Agent: Handles developer questions with MCP tools
  • Billing Agent: Answers billing questions using vector store
  • Administration Agent: Manages admin requests with A2A protocol
